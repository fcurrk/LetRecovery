# 系统备份

**系统备份**页用于把一个分区捕获成镜像文件。

## 格式

| 格式 | 适用 | 压缩 |
| --- | --- | --- |
| **WIM** | 标准格式，兼容性好（推荐）。 | LZX |
| **ESD** | 压缩率更高，文件更小。 | LZMS（solid） |
| **SWM** | 分卷存储（先捕获为 WIM 再切分），便于传输。 | LZX，按大小切分 |
| **GHO** | Ghost 格式（需要 Ghost 引擎）。 | Ghost 自有 |

选 **SWM** 时需填**分卷大小**（单位 MB，有效范围 **512–8192**）。

## 步骤

1. 选择要备份的**源分区**。
2. 选择**格式**与**保存位置**。
3. 填写**名称**（必填）与**描述**（可选）。
4. 需要时勾选**增量备份**（见下）。
5. 点击**开始备份**。

::: tip 名称是必填项
源分区、保存位置、名称三者都填好后，"开始备份"按钮才会可用；描述则可留空。
:::

## 增量备份

"增量备份（追加到现有镜像）"是一个可勾选项，行为如下：

- 通过**浏览…**选中一个**已存在**的镜像文件时，该选项会被**自动勾选**；你也可以手动勾选。
- 增量模式下，LetRecovery 会向现有镜像文件里**追加一个新的镜像索引（INDEX）**，
  而**不是**重写整个文件，也不是新增"分卷"。
- 追加仅对 **WIM / ESD** 有效；SWM 与 GHO 每次都是全新捕获。

::: tip 增量 ≠ 分卷
"分卷"是 SWM 把一个镜像切成多块的概念；"增量"是在同一个 WIM/ESD 里多存一个版本（索引）。
二者不要混淆。
:::

## 备份正在运行的系统

**当前系统分区**在使用中无法可靠地原地备份，所以 LetRecovery 会**经 WinPE**完成：
先准备 PE 引导、把这次备份的名称/格式/增量等选项写入一个配置文件，重启进入 WinPE 后，
PE 端读取该配置再捕获镜像。备份**非系统**分区（或在 PE 里运行）则直接进行。

::: tip 自动校验
PE 端在捕获/部署前会先校验镜像完整性，损坏镜像会及早失败，避免白做一遍。
:::
