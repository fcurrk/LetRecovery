# 系统安装

**系统安装**页用于把 Windows 镜像部署到某个分区。

## 支持的镜像格式

| 格式 | 说明 |
| --- | --- |
| **WIM / ESD / SWM** | 标准 Windows 镜像，经 wimlib 释放（SWM 为分卷，会自动识别同目录的其余分卷）。 |
| **GHO** | Ghost 镜像，经内置 Ghost 引擎还原。 |
| **ISO** | 自动挂载，使用其中的 `install.wim` / `install.esd` / `install.swm`。 |
| **XP / 2003 i386** | 原版 XP/2003 安装盘（根目录是 `\I386`、没有 `install.wim`）会被识别为文本模式安装介质，详见 [Windows XP / 2003 安装](/guide/xp-install)。 |

选好镜像后，选择要安装的**版本**（镜像分卷索引）。版本下拉里会自动过滤掉不可安装的卷
（如 WindowsPE / Setup 介质卷），默认选中第一个可安装版本。

## 选择目标分区

在列表里选目标分区。表格会显示容量、可用空间、卷标、分区表（GPT/MBR）、BitLocker
状态，以及该分区是否已有系统。

::: tip 目标盘被 BitLocker 加密了？
若目标或数据分区处于锁定状态，安装前会弹出解锁对话框。对加密的**系统盘**，LetRecovery
会优先把恢复密钥透传进 PE 解锁（快），拿不到密钥时才回退彻底解密——详见
[BitLocker 加密盘重装](/guide/bitlocker)。
:::

## 安装方式

LetRecovery 会自动决定怎么装：

- **直接模式**——目标**不是**正在运行的系统盘，或 LetRecovery **本来就在 WinPE 里**
  运行时。直接格式化分区并释放镜像。
- **经 PE 模式**——从桌面把系统装到**正在运行的系统盘**时。LetRecovery 写一个临时
  WinPE 引导项并重启进入，再由 [PE 端](/guide/what-is-letrecovery#工作原理)格式化
  C: 并释放镜像。运行中的系统无法覆盖自身，所以要重启。

## 部署前镜像校验

在**经 PE 模式**与 **PE 端**安装流程里，格式化目标盘**之前**会先校验镜像完整性
（WIM/ESD 经 wimlib）。坏镜像会在**动盘之前**被拦下，下错/下坏的镜像不会糟蹋你的数据。

::: warning 直接模式与 GHO 不在此列
- GHO 镜像只做头/尾结构性检查，不走 wimlib 校验。
- 从桌面**直接**装到非系统盘的"直接模式"目前不在部署前自动校验范围内。这种情况
  建议先用[工具箱 → 镜像校验](/guide/toolbox#镜像与完整性)手动校验一遍再装。
:::

## 引导模式

LetRecovery 会自动识别 UEFI / Legacy（GPT→UEFI、MBR→Legacy）并写入对应引导文件。
需要时也可在**引导模式**下拉框里手动指定 `自动 / UEFI / Legacy`。该下拉框始终可见；
启用无人值守时它与"自定义无人值守"按钮并列同一行，否则单独成行。

## 自定义分区脚本（进阶）

启用[高级选项](/guide/advanced-options)后，会多出一个"运行 Diskpart 脚本"选项：
把 `.cmd` / `.bat` / `.txt` 脚本放进 `<程序目录>\diskpart\`，安装会在**格式化之前**执行它们
（脚本失败则中止安装）。适合需要特殊分区布局的场景。

## 驱动与选项

开始前可在[高级选项](/guide/advanced-options)里启用驱动导出/导入、磁盘控制器驱动
注入、无人值守、注册表优化、WiFi 配置迁移等。Win7 / XP 镜像还会自动显示各自的
兼容性开关。

::: danger
安装会**格式化目标分区**，请先备份。
:::
